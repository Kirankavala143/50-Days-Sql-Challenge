Day 6: Data Cleaning – Outlier Detection & Handling

Objective
Identify, analyze, and handle outliers in the dataset while preserving meaningful business information.

Tasks Completed

Detected salary outliers using predefined business thresholds.
Added an is_outlier flag to classify extreme values.
Differentiated between valid outliers (e.g., CEO salary) and invalid outliers (data errors).
Corrected incorrect outlier values using client-provided reference data.
Preserved legitimate extreme values instead of blindly removing them.
Validated the cleaned dataset to ensure accuracy and consistency.

Key Learning

Not all outliers indicate errors; some represent valid business scenarios.
Outliers should be investigated before taking corrective action.
Business context is critical when deciding how to handle extreme values.
Data cleaning is about improving quality without losing important information.

Outcome
Successfully identified, classified, and handled outliers, resulting in a cleaner and more reliable dataset while preserving meaningful insights.
