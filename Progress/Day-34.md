📅 Day 34 – SQL 50-Day Challenge

Today, I learned how Indexes improve query performance by reducing the time required to search data in large tables.

📌 Topics Covered
Creating an Index on a single column (emp_id)
Creating an Index on another column (dept_id)
Creating a Composite Index (emp_id, salary_date)
Viewing all indexes using SHOW INDEX
💻 Queries Practiced
Create Index on emp_id
Create Index on dept_id
Create Composite Index
Display Existing Indexes
📚 Key Learnings
Indexes significantly improve the speed of SELECT queries.
Composite indexes are useful when filtering using multiple columns.
SHOW INDEX helps inspect all indexes present on a table.
Too many indexes can slow down INSERT, UPDATE, and DELETE operations.
