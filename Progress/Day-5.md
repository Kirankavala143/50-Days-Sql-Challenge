Day 5: Data Cleaning – Handling Invalid Values
Objective

Identify and correct logically invalid values within the dataset to improve data quality and reliability.

Tasks Performed
Identified invalid records, including negative salaries, incorrect age values, and out-of-range ratings.
Applied business rules to determine valid data ranges.
Corrected invalid values using client-provided reference data.
Implemented SQL CASE WHEN statements for bulk updates.
Verified data consistency after corrections.
Key Learnings
Invalid data is not always missing; it can also be logically incorrect.
Data corrections should be based on trusted sources and business requirements.
Understanding the business context is essential for effective data cleaning.
Outcome

Successfully improved dataset accuracy and consistency by removing logically invalid values and ensuring compliance with defined business rules.
