# 50 Days SQL Challenge 🚀

## Day 3: Data Cleaning (Handling Missing Values)

### Objective

Clean and improve data quality by handling missing values across multiple tables while preserving the original raw dataset.

### Tasks Completed

- Created cleaned versions of existing tables
- Identified NULL values in employee records
- Identified empty string values across multiple columns
- Converted empty values into NULL values for consistency
- Replaced missing values using appropriate defaults
- Preserved original raw tables for future reference
- Verified cleaned data after updates
- Prepared dataset for further transformation and analysis

### SQL Concepts Used

- UPDATE
- WHERE
- IS NULL
- CASE
- Data Cleaning Techniques
- Data Validation

### Key Learnings

- NULL values and empty strings are different and must be handled separately
- Data cleaning should never be performed directly on raw datasets
- Business rules play an important role when replacing missing values
- Consistent data improves analysis accuracy and reliability

### Outcome

Successfully cleaned missing values and improved dataset consistency across multiple tables. The dataset is now better prepared for future data transformation and analysis tasks.
